# kaifa-skill
# kaifa-skill  > 一个以“提高开发者参与度”为核心的协作式开发 Skill 系统  `kaifa-skill` 不把开发过程交给 LLM 黑箱完成，而是要求它：  - 显式说明当前所处状态 - 在关键节点给出推荐方案和备选项 - 用“选择题”形式让开发者拍板 - 按阶段推进：调查、业务设计、详细设计、编码验证、严格审查、收尾归档 - 沉淀项目理解报告、业务设计、编码报告和历史回顾  它的核心不是“自动写代码”，而是：  > 让 LLM 负责整理方案，让开发者负责关键决策。  适合： - 中大型功能开发 - 已有项目二开 - 复杂需求分析 - 需要设计、验证、审查和文档沉淀的开发任务  不适合： - 一次性小脚本 - 纯查询问题 - 不需要设计和归档的简单修改
